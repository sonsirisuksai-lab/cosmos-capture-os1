## 🧠 TIER S — สร้างก่อน (หัวใจของระบบ)

### 1. Crew Orchestrator (`src/lib/ai/orchestrator.ts`)

สร้างระบบผู้บัญชาการ AI Agents:

**ฟังก์ชันที่ต้องมี:**
- `routeAgent(input)` — เลือก Agent ที่เหมาะสม
- `orchestrate(request)` — รับคำขอ → เลือก Agent → ส่ง response
- `orchestrateWithAI(input, apiKey)` — ใช้ AI จริง (Optional)
- `maka(prompt, apiKey)` — Maka Prompt Mode (โยนอะไรก็ได้!)

**Agents (10 ตัว):**
| Agent | Role | Domain |
|-------|------|--------|
| Luffy 🏴‍☠️ | Vision | goal, dream, purpose, why |
| Zoro ⚔️ | Execution | code, build, debug, logic |
| Nami 🗺️ | Finance | budget, plan, resource, cost |
| Robin 📚 | Knowledge | research, learn, history, insight |
| Franky 🔧 | Builder | build, system, automate, pipeline |
| Chopper 🩺 | Health | wellness, sleep, exercise, mental |
| Brook 🎵 | Memory | nostalgia, story, emotion, past |
| Jinbe ⚓ | Operations | routine, discipline, process, daily |
| Sanji 🍳 | Quality | review, security, audit, test |
| Usopp 🎯 | Social | people, communication, strategy |

**โค้ดที่ต้องมีใน orchestrator.ts:**
- `Agent`, `AgentRole`, `AGENTS` (map ทั้ง 10 ตัว)
- `OrchestratorRequest`, `OrchestratorResponse` interfaces
- `routeAgent()` — routing logic
- `orchestrate()` — main function
- `orchestrateWithAI()` — AI integration (Optional)
- `maka()` — Maka Prompt interface
- `generateMockResponse()` — fallback เมื่อไม่มี API
- `generateSuggestions()` — generate follow-up suggestions

---

### 2. Knowledge Evolution Engine (`src/lib/knowledge/evolution.ts`)

สร้างระบบที่ทำให้ความรู้ "โต" เอง:

**ฟังก์ชันที่ต้องมี:**
- `evolveKnowledge()` — ตรวจสอบและปรับปรุง knowledge objects
- `mergeNotes(note1, note2)` — รวม note ที่ซ้ำกัน
- `splitNote(note)` — แยก note ที่ใหญ่เกิน
- `promoteKnowledge(note)` — ยกระดับ note ที่สำคัญ
- `archiveDeadKnowledge()` — ย้าย note ที่ตายแล้วไป archive
- `detectKnowledgeDecay()` — ตรวจจับความรู้ที่กำลัง decay
- `calculateConfidence(note)` — คำนวณ confidence score
- `calculateRelevance(note, context)` — คำนวณ relevance

**สัญญาณที่ใช้:**
- `confidence` — ความมั่นใจในข้อมูล
- `relevance` — ความเกี่ยวข้องกับบริบทปัจจุบัน
- `freshness` — ความใหม่ของข้อมูล
- `importance` — ความสำคัญ
- `reuseCount` — จำนวนครั้งที่ถูกใช้
- `reviewCount` — จำนวนครั้งที่ถูก review

---

### 3. Graph Analytics (`src/lib/graph/analytics.ts`)

สร้างระบบอ่านสุขภาพเครือข่ายข้อมูล:

**ฟังก์ชันที่ต้องมี:**
- `getNodeImportance()` — หา node ที่สำคัญที่สุด
- `findHiddenClusters()` — หา cluster ที่ซ่อนอยู่
- `detectKnowledgeGap()` — ตรวจจับช่องว่างของความรู้
- `findOrphanNodes()` — หา node ที่โดดเดี่ยว
- `calculateDensity()` — คำนวณความหนาแน่นของ graph
- `findEmergingTopics()` — หาหัวข้อที่กำลังมา
- `getCentrality(nodeId)` — คำนวณ centrality
- `getConnectivity()` — วัดความเชื่อมต่อของระบบ

**Metrics:**
- centrality
- connectivity
- density
- orphan rate
- cluster growth
- bridge nodes

---

### 4. Digital Twin Foundation (`src/lib/twin/foundation.ts`)

สร้างรากฐานของ Digital Twin:

**ฟังก์ชันที่ต้องมี:**
- `buildTwinProfile()` — สร้าง profile จากข้อมูลทั้งหมด
- `updateTwinMetrics()` — อัปเดต metrics
- `getTwinState()` — ดึงสถานะปัจจุบันของ Twin
- `captureTwinMemory()` — บันทึก memory ของ Twin
- `getTwinBehavior()` — วิเคราะห์พฤติกรรม

**Twin Components:**
- `TwinProfile` — ตัวตน (ชื่อ, อายุ, บทบาท)
- `TwinMetrics` — ตัวเลขสุขภาพระบบ
- `TwinState` — สถานะปัจจุบัน
- `TwinMemory` — ความทรงจำของ Twin
- `TwinBehavior` — รูปแบบพฤติกรรม
